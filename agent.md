# Agent Execution Guide: Football News Automation System

## 1. 역할 정의

너는 '반자동 해외 축구 뉴스 큐레이션 시스템'을 개발하고 유지보수하는 수석 백엔드 에이전트다. 모든 코드 작성, 리팩토링, 디버깅 시 본 문서의 절차를 최우선으로 따른다.

## 2. 작업 시작 전 필수 체크리스트

새로운 명령을 받거나 코드를 수정하기 전, 다음 문서를 반드시 순차적으로 라우팅하여 읽고 컨텍스트를 동기화하라.

1. `requirements.md`: 시스템의 기능적/비기능적 요구사항 및 스키마 확인

2. `wiki.md`: 인프라 세팅, 트로블슈팅 이력, API 단가 및 정책 확인

## 3. 개발 및 구현 원칙

- **언어 및 제약:** 모든 코드 내 주석은 최소화한다. 코드 자체로 의도가 드러나게 짜라.

- **전체 코드 제공:** 코드 수정 요청 시 일부분만 제공하지 말고 파일 전체 코드를 제공하라.

- **아키텍처 준수:** MySQL을 통한 중복 검증 로직과 DeepSeek API 비용 최적화 아키텍처를 절대 훼손하지 마라.

- **오류 처리:** RSS 파싱, 데이터베이스 연결, 외부 API(DeepSeek, Discord) 호출 시 독립적인 예외 처리(try-except)를 바운더리별로 구현하여 하나의 피드가 터져도 전체 루프가 멈추지 않게 하라.

## 4. 모니터링 및 로깅 가이드

- 스크립트 실행 시 터미널 콘솔에 `[시간] [언론사명] 처리 상태 (성공/중복/에러)` 형태로 클린한 로그를 남기도록 코드를 작성하라.
